Planet rendering options

This dialog provides extended options for fine-tuning the way SpaceXpanse loads and displays textures for planetary surfaces. For basic render options, see also Visual effects.

Note that some options in this dialog may be overridden by graphics driver settings. This can be avoided by setting the appropriate driver parameters to "application-controlled" or equivalent.

Surface texture load options

Load on demand: Load textures during the simulation only when they are needed. This is the recommended setting, in particular if high-resolution textures are installed. It reduces the load time and memory requirements compared to the pre-load option.

Max. load frequency: The maximum number of texture patches loaded per second. Increasing this value speeds up the load process and avoids momentary low-resolution rendering when the camera pans to a new area, but it may lead to occasional drops in frame rate (stuttering) during the simulation.

Pre-load at session start: All textures for all planets are loaded when a simulation session is launched. This option can greatly increase load times and texture memory requirements, and should generally be avoided.

Mipmap interpolation

Mipmap interpolation can improve the visual appearance of planet textures by avoiding pixel flickering during movement. It may affect frame rates on some systems.

Do not load mipmaps: Disables mipmap interpolation (lowest visual quality, highest performance).

Point sampling: Uses mipmaps, but does not interpolate them (medium visual quality and performance).

Linear interpolation: Uses interpolated mipmaps (highest visual quality, lowest performance).

Mipmap bias: If mipmaps are enabled, this slider allows to adjust the mipmap level-of-detail (LOD) bias. Sharper uses higher resolution mipmaps, which give a sharper appearance, but can induce flickering. Smoother uses lower lower resolution mipmaps which look smoother and produce less flickering.

Anisotropic filtering

Anisotropic filtering can improve the visual appearance of surfaces viewed at oblique angles. In general, it makes planetary surface features appear sharper towards the horizon.

Filter level: Amount of filtering. None disables anisotropic filtering. Higher filter levels may affect simulation frame rates.
