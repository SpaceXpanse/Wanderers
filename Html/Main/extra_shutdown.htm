Shutdown options

The settings in this dialog determine how SpaceXpanse behaves after a simulation session is closed.

  • De-allocate memory and display launchpad dialog. This is the standard behaviour: SpaceXpanse closes the simulation session in an orderly fashion, and returns to the Launchpad dialog.
  • Respawn SpaceXpanse process. With this option enabled, SpaceXpanse skips most of the cleanup code at the end of the simulation session, and simply kills the running SpaceXpanse process. A new process is launched to replace the old one. This can be substantially faster than the standard method, in particular if a large number of textures had been loaded. However, it is "unclean" and may prevent some modules from saving their exit state
  • Terminate SpaceXpanse process. This option simply kills the SpaceXpanse process altogether. The user has to restart SpaceXpanse for a new simulation session.

Notes

  • Use of the "standard" method of deallocating memory is encouraged, because the other options may prevent addon modules from terminating cleanly.
  • The "respawn" and "terminate" options may sometimes bypass bugs in the cleanup or initialisation code of badly written addons that cause errors in the standard method.
  • For this reason, using the "respawn" and "terminate" options is strongly discouraged for addon developers during testing and debugging of their code, since it may mask existing problems with the code.
  • Addon developers should choose the standard option and make sure that their modules behave correctly in multiple simulation sessions without quitting SpaceXpanse.
